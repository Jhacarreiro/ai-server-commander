# Code of Conduct

## Our standard

Participants are expected to be respectful, constructive and focused on improving the project. Examples of positive behavior include:

- giving specific, technically grounded feedback;
- acknowledging mistakes and correcting them;
- respecting different experience levels and perspectives;
- protecting user privacy and deployment secrets;
- disagreeing with ideas without attacking people.

Unacceptable behavior includes harassment, discrimination, threats, deliberate disclosure of private information, sustained disruption, or publishing another person's credentials or production data.

## Scope

This standard applies to repository issues, pull requests, discussions, code review and other project spaces.

## Enforcement

Report conduct concerns privately through the repository owner's GitHub profile. Maintainers may edit or remove content, close discussions, reject contributions or restrict participation when necessary to protect the project and its community.

Reports will be handled as confidentially as practical. Retaliation against someone who raises a good-faith concern is not acceptable.
